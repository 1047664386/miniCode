## 两个 Bug 修复

### 1. Electron 打包版登录/注册不工作
- 根因：`electron-bridge.ts` 的 `isApiPath()` 只代理 `/api/` 和 `/health`，不代理 `/cloud-api/`
- 前端 `cloudFetch()` base 是 `/cloud-api`，在 `file://` 协议下变成 `file:///cloud-api/...` → Failed to fetch
- 后端 server-cloud (4000端口) 的登录/注册 API 本身工作正常
- 修复方案：`isApiPath` 加 `/cloud-api/` 匹配 + proxy 到 4000 端口

### 2. 无法中止正在执行的 AI 回复
- 根因：前端没有 stop 按钮。`running=true` 时发送按钮 disabled（只显示 spinner），用户无法中断
- 后端已支持 abort（`req.on('close')` → `abort.abort()` → `runAgent` 收到 signal 停止）
- 修复：ChatPanel.tsx 添加 `chatAbortRef` + `stopChat()`，running 时发送按钮变红色方块 stop 按钮
- fetch 传 `signal: chatAbort.signal`，AbortError 时优雅提示"已停止生成"

## Bug 修复：文件树不实时刷新

### 根因
1. 后端 chokidar (IndexWatcher) 监听文件变更后只更新索引，不通知前端
2. 前端 TreeNode 组件不监听 treeVersion 变化，已展开的子目录不刷新
3. ChatPanel 的 finally 块虽然调用 loadTree('.') 但只刷新根目录
4. acceptPending 后不刷新文件树

### 修复方案（4个文件）
1. **watcher.ts**：WatcherOptions 新增 onFileChange 回调，flush 完成后批量通知变更事件（保留 add/change/unlink 原始类型）
2. **main.ts**：新增 `GET /api/fs/events` SSE 端点，桥接 chokidar → SSE；watcher 初始化时传入 onFileChange 广播给 SSE 客户端
3. **FileTree.tsx**：TreeNode 订阅 treeVersion 变化自动 refreshChildren；FileTree 组件建立 EventSource 连接到 /api/fs/events，收到 fs_change 事件后 bumpTree + loadTree
4. **store.ts**：acceptPending 添加 loadTree('.') + bumpTree() 调用
5. **server-node 同步修改**：services.ts 添加 fsEventClients + onFileChange，register.ts 添加 /api/fs/events SSE 端点
