## Bug 修复：Electron 登录后历史模块 401 + 前端崩溃

### 根因分析
1. **401 原因**：Electron `file://` 协议是独立 opaque origin，到 `http://127.0.0.1:4000` 的 fetch 是跨站请求。server-cloud 的 JWT cookie 设置了 `SameSite=Lax`，不会被跨站 fetch 发送 → `mci_jwt` cookie 丢失 → 401
2. **崩溃原因**：`loadSessions()` 未检查 `r.ok`，401 响应体 `{error:'unauthorized'}` 被直接赋给 `sessionList`，`sessionList.map()` 报 `t.map is not a function`

### 修改
1. **server-cloud/routes/auth.ts**：login/register 响应体增加 `token` 字段（原来只在 Set-Cookie 里）
2. **desktop/src/store.ts**：
   - 新增 `authToken` 状态，从 localStorage 持久化（`mci.authToken`）
   - `sessionFetch` 和 `cloudFetch` 自动注入 `Authorization: Bearer <token>` header
   - `loadSessions()` 增加防御性编码：检查 `r.ok`、验证 `Array.isArray(list)`，401 时保持原值不崩溃
   - login/register 后保存 token 到 localStorage
   - logout 后清除 token
3. **agents-window/store.ts**：已自带 `r.ok` 检查（无需修改），共享 `sessionFetch` 的 Authorization header 修复

## 历史会话模块 Bug 修复（第二轮）

### 问题 1：新会话名称不自动取首条消息前 40 字符
- **根因**：后端 `SessionStore.append()` 已有自动标题逻辑（首条 user 消息前 40 字符），但前端发消息完成后没有调用 `loadSessions()` 刷新 sessionList
- **修复**：
  - `ChatPanel.tsx`：finally 块中增加 `loadSessions().catch(() => {})`，解构添加 `loadSessions`
  - `AgentsMain.tsx`：finally 块中增加 `useAgentsStore.getState().loadSessions().catch(() => {})`

### 问题 2：选中历史会话不拉取历史消息
- 401 问题已在上一轮修复（Authorization header）
- agents-window 的 `AgentsMain` 有 `useEffect([activeId])` 拉取历史消息，逻辑正确

### 问题 3：switchSession/resumeSession 消息字段丢失（BUG-1/2）
- **根因**：`switchSession` 和 `resumeSession` 只保留 `role`/`content`，丢失了后端已有的 `toolName`/`pendingEditId`/`pendingEditPath`/`ts`
- **修复**：映射后端字段 + 展开 `uiMeta`
  - `m.toolName → _toolName + _toolRole: 'result'`
  - `m.pendingEditId → pendingEditId`
  - `m.pendingEditPath → pendingEditPath`
  - `m.uiMeta → ...展开`

### 问题 4：前端 ChatMsg 扩展字段无法持久化（BUG-3）
- **根因**：`_toolRole`/`_toolArgs`/`_thinkingMs` 等字段只存在于前端内存，后端 `SessionMessage` 不存储
- **修复**：
  - `SessionMessage` 类型添加 `uiMeta?: Record<string, unknown>` 字段
  - Chat handler 中 tool_call 事件时 `sessions.append({ role: 'tool', ..., uiMeta: { _toolRole, _toolName, _toolArgs } })`
  - PgStorage 的 `messageFromRow` 和 `append` 支持 `uiMeta`
  - Prisma schema Message model 添加 `uiMeta Json?` 字段
  - 前端 `switchSession`/`resumeSession`/`AgentsMain useEffect` 从 `uiMeta` 恢复扩展字段
