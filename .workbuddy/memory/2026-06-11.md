# 2026-06-11 工作日志

## Bug 修复：Electron 登录后历史模块 401 + 前端崩溃

### 根因分析
1. **401 原因**：Electron `file://` 协议是独立 opaque origin，到 `http://127.0.0.1:4000` 的 fetch 是跨站请求。server-cloud 的 JWT cookie 设置了 `SameSite=Lax`，不会被跨站 fetch 发送 → `mci_jwt` cookie 丢失 → 401
2. **崩溃原因**：`loadSessions()` 未检查 `r.ok`，401 响应体 `{error:'unauthorized'}` 被直接赋给 `sessionList`，`sessionList.map()` 报 `t.map is not a function`

### 修改
1. **server-cloud/routes/auth.ts**：login/register 响应体增加 `token` 字段（原来只在 Set-Cookie 里）
2. **desktop/src/store.ts**：
   - 新增 `authToken` 状态，从 localStorage 持久化（`mci.authToken`）
   - `sessionFetch` 和 `cloudFetch` 自动注入 `Authorization: Bearer <token>` header
   - `loadSessions()` 增加防御性编码：检查 `r.ok`、验证 `Array.isArray(list)`，401 时保持原值不崩溃
   - login/register 后保存 token 到 localStorage
   - logout 后清除 token
3. **agents-window/store.ts**：已自带 `r.ok` 检查（无需修改），共享 `sessionFetch` 的 Authorization header 修复
